# 🚀 Bicep Deep-Dive Notes For Terraform Engineers

> Learn Bicep by Relating Every Concept with Terraform  
> Goal: Think in Bicep Like You Already Think in Terraform

---

# 🧠 CORE UNDERSTANDING

Before learning syntax, understand the REAL difference.

| Terraform | Bicep |
|---|---|
| Generic IaC Tool | Azure Native IaC Tool |
| Works with multiple clouds | Works only with Azure |
| Uses Providers | Directly uses Azure Resource Types |
| Uses Terraform Engine | Uses Azure ARM Engine |
| Maintains `.tfstate` | Azure manages deployment state |
| HCL Language | Bicep DSL |
| Provider downloads APIs | Azure APIs directly available |

---

# 🔥 MOST IMPORTANT REALIZATION

## Terraform

```text
Terraform → Provider → Azure API
```

---

## Bicep

```text
Bicep → ARM Template → Azure API
```

---

# 🧠 HOW TERRAFORM THINKS

Terraform internally:

```text
1. Read .tf files
2. Download provider
3. Create dependency graph
4. Compare with tfstate
5. Create execution plan
6. Apply changes
```

---

# 🧠 HOW BICEP THINKS

Bicep internally:

```text
1. Read .bicep file
2. Convert to ARM JSON
3. Send directly to Azure ARM
4. Azure handles deployment
```

---

# ⚡ BIGGEST DIFFERENCE

| Terraform | Bicep |
|---|---|
| Terraform manages infrastructure state | Azure manages deployment state |
| Requires backend configuration | No backend needed |
| State locking required | No state locking |
| Drift detection via tfstate | ARM handles resource state |

# 1️⃣ VARIABLES vs PARAMETERS

---

# 🧠 TERRAFORM THINKING

```hcl
variable "location" {
  type = string
  default = "East US"
}
```

Terraform says:

```text
"Take input from user during deployment"
```

---

# 🧠 BICEP THINKING

```bicep
param location string = 'East US'
```

Bicep says:

```text
"Azure deployment input"
```

---

# 🔥 REAL DIFFERENCE

| Terraform | Bicep |
|---|---|
| variable block | param keyword |
| type defined separately | inline type |
| verbose syntax | compact syntax |

---

# 📌 TERRAFORM

```hcl
variable "environment" {
  type = string
}
```

---

# 📌 BICEP

```bicep
param environment string
```

---

# 🧠 MEMORY TRICK

```text
Terraform variable = Bicep param
```

|---|---|

| bool | bool |

# 2️⃣ LOCALS vs VARIABLES

---

# 🧠 TERRAFORM THINKING

```hcl
locals {
  storage_name = "${var.env}storage"
}
```

Used for:
- calculations
- reusable naming
- tagging

---

# 🧠 BICEP THINKING

```bicep
var storageName = '${env}storage'
```

---

# 🔥 REAL DIFFERENCE

| Terraform | Bicep |
|---|---|
| locals block | var keyword |
| multiple locals inside block | one var per declaration |
| interpolation `${}` | same interpolation |

---

# 📌 TERRAFORM

```hcl
locals {
  rg_name = "dev-rg"
}
```

---

# 📌 BICEP

```bicep
var rgName = 'dev-rg'
```

---

# 🧠 IMPORTANT

In Bicep:

```text
var = internal calculated value
```

NOT deployment input.

# 3️⃣ RESOURCE CREATION

This is the HEART of Bicep.

---

# 🧠 TERRAFORM THINKING

```hcl
resource "azurerm_storage_account" "stg" {
  name = "mystorage"
}
```

Terraform:
- uses provider plugin
- provider knows APIs

---

# 🧠 BICEP THINKING

```bicep
resource stg 'Microsoft.Storage/storageAccounts@2025-06-01' = {
  name: 'mystorage'
}
```

Bicep:
- directly uses Azure Resource Type
- explicitly uses API version

---

# 🔥 BIGGEST DIFFERENCE

| Terraform | Bicep |
|---|---|
| Provider handles API versions | You specify API version |
| Provider-specific resource names | Azure-native resource types |
| HCL block syntax | JSON-like object syntax |

---

# 📌 TERRAFORM BREAKDOWN

```hcl
resource "azurerm_storage_account" "stg"
```

| Part | Meaning |
|---|---|
| resource | keyword |
| azurerm_storage_account | provider resource |
| stg | local name |

---

# 📌 BICEP BREAKDOWN

```bicep
resource stg 'Microsoft.Storage/storageAccounts@2025-06-01'
```

| Part | Meaning |
|---|---|
| resource | keyword |
| stg | symbolic name |
| Microsoft.Storage | provider namespace |
| storageAccounts | resource type |
| 2025-06-01 | API version |

---

# 🔥 MOST IMPORTANT BICEP THING

## THIS PART:

```bicep
'Microsoft.Storage/storageAccounts@2025-06-01'
```

must be remembered.

---

# 🧠 HOW TO THINK IN BICEP

Whenever creating any Azure resource:

```text
1. Find resource type
2. Find API version
3. Add properties
```

---

# 📌 Example

## Terraform

```hcl
resource "azurerm_resource_group" "rg" {
  name = "dev-rg"
  location = "East US"
}
```

---

## Bicep

```bicep
resource rg 'Microsoft.Resources/resourceGroups@2025-04-01' = {
  name: 'dev-rg'
  location: 'East US'
}
```

# 5️⃣ DEPENDENCIES

This is VERY important.

---

# 🧠 TERRAFORM THINKING

Terraform builds dependency graph.

Sometimes explicit dependency needed:

```hcl
depends_on = []
```

---

# 🧠 BICEP THINKING

Dependencies mostly automatic.

---

# 📌 Example

```bicep
resource stg 'Microsoft.Storage/storageAccounts@2025-06-01' = {
  name: 'mystorage'
}

resource container 'Microsoft.Storage/storageAccounts/blobServices/containers@2025-06-01' = {
  name: '${stg.name}/default/container'
}
```

---

# 🧠 WHAT BICEP UNDERSTANDS

```text
container uses stg.name
→ storage account must exist first
```

Automatic dependency.

---

# 🔥 REAL DIFFERENCE

| Terraform | Bicep |
|---|---|
| Explicit depends_on sometimes needed | Mostly automatic |
| Graph built from references | ARM handles dependency graph |

# 6️⃣ MODULES

---

# 🧠 TERRAFORM THINKING

```hcl
module "network" {
  source = "./modules/network"
}
```

Reusable infrastructure package.

---

# 🧠 BICEP THINKING

```bicep
module network './modules/network.bicep' = {
  name: 'networkDeploy'
}
```

---

# 🔥 DIFFERENCE

| Terraform | Bicep |
|---|---|
| source | file path |
| module name | symbolic name |
| variables | params |

---

# 📌 TERRAFORM

```hcl
module "network" {
  source = "./modules/network"

  location = "East US"
}
```

---

# 📌 BICEP

```bicep
module network './modules/network.bicep' = {
  name: 'networkDeploy'

  params: {
    location: 'East US'
  }
}
```

---

# 🧠 IMPORTANT

Bicep modules deploy as:
```text
Nested ARM deployments
```

# 🔟 STATE MANAGEMENT

THIS IS HUGE.

---

# 🧠 TERRAFORM THINKING

Terraform MUST track:
- created resources
- resource mapping
- changes

using:

```text
terraform.tfstate
```

---

# 🧠 BICEP THINKING

Azure ARM already knows infrastructure state.

So:

```text
No state file required.
```

---

# 🔥 DIFFERENCE

| Terraform | Bicep |
|---|---|
| tfstate mandatory | no state |
| backend required | no backend |
| remote state management | ARM deployment history |
| state locking needed | no locking |
