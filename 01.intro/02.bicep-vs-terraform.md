# 🚀 Terraform vs Bicep — Complete Comparison Notes

> Learn Bicep Easily By Relating It With Terraform

---

# 📘 First Understand The Reality

If you already know Terraform:

```text
You already understand Infrastructure as Code.
```

That means:
- variables
- modules
- loops
- dependencies
- outputs
- environments
- deployments

all concepts are already known to you.

Now in Bicep:

```text
Only syntax and Azure-native structure changes.
```

---

# 🧠 Biggest Mindset Difference

| Terraform | Bicep |
|---|---|
| Multi-cloud tool | Azure-only tool |
| Uses providers | Directly talks to Azure |
| Uses HCL language | Uses Bicep language |
| Needs state file | No state file |
| Terraform manages infra state | Azure manages deployment state |

---

# 🔥 Real Meaning

## Terraform Thinking

```text
Use provider plugins to create infrastructure
```

---

## Bicep Thinking

```text
Directly describe Azure infrastructure to Azure
```

---

# 1️⃣ VARIABLES vs PARAMETERS

---

# ✅ Terraform

```hcl
variable "location" {
  default = "East US"
}
```

---

# ✅ Bicep

```bicep
param location string = 'East US'
```

---

# 🧠 Real Meaning

Both are taking input values during deployment.

---

# 📌 Think Like This

| Terraform | Bicep |
|---|---|
| variable | param |

---

# 🧠 Memory Trick

```text
Terraform variable = Bicep param
```

---

# 🔥 Main Difference

Terraform:

```hcl
variable "location" {
  type = string
}
```

Bicep:

```bicep
param location string
```

Bicep is shorter and cleaner.

---

# 2️⃣ LOCALS vs VARIABLES

---

# ✅ Terraform

```hcl
locals {
  storage_name = "devstorage"
}
```

---

# ✅ Bicep

```bicep
var storageName = 'devstorage'
```

---

# 🧠 Real Meaning

Used for:
- calculations
- reusable values
- naming conventions

---

# 📌 Think Like This

| Terraform | Bicep |
|---|---|
| locals | var |

---

# 📌 Example

Terraform:

```hcl
locals {
  unique_name = "${var.prefix}-${random_id.id.hex}"
}
```

---

Bicep:

```bicep
var uniqueName = '${prefix}${uniqueString(resourceGroup().id)}'
```


---

# 🧠 Understand Bicep Structure

```bicep
resource stg 'Microsoft.Storage/storageAccounts@2025-06-01'
```

---

# Breakdown

| Part | Meaning |
|---|---|
| resource | keyword |
| stg | symbolic name |
| Microsoft.Storage | provider |
| storageAccounts | resource type |
| 2025-06-01 | API version |

---

# 🔥 IMPORTANT

In Terraform:

```text
Provider handles API versions internally
```

In Bicep:

```text
You explicitly define API version
```

# 7️⃣ DEPENDENCIES

---

# ✅ Terraform

```hcl
depends_on = []
```

---

# ✅ Bicep

Mostly automatic.

---

# 📌 Example

```bicep
resource stg 'Microsoft.Storage/storageAccounts@2025-06-01' = {
  name: 'mystorage'
}

resource container 'Microsoft.Storage/storageAccounts/blobServices/containers@2025-06-01' = {
  name: '${stg.name}/default/container'
}
```

---

# 🧠 What Happened?

Bicep automatically understands:

```text
Container depends on Storage Account
```

---

# 🔥 Real Difference

Terraform:
Sometimes explicit dependency needed.

Bicep:
Dependencies mostly inferred automatically.

---

# 8️⃣ COUNT / FOR_EACH vs LOOPS

---

# ✅ Terraform

```hcl
count = 3
```

---

# ✅ Bicep

```bicep
[for i in range(0,3): {

}]
```

---

# 📌 Example

Terraform:

```hcl
resource "azurerm_storage_account" "stg" {
  count = 3
}
```

---

Bicep:

```bicep
resource stg 'Microsoft.Storage/storageAccounts@2025-06-01' = [
  for i in range(0,3): {
    name: 'storage${i}'
  }
]
```

---

# 🧠 Real Meaning

Create multiple resources dynamically.

---

# 📌 Think Like This

| Terraform | Bicep |
|---|---|
| count | for loop |
| for_each | for loop |

---

- deployed resources
- mapping
