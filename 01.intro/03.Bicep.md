# 🚀 Bicep Complete Notes For Terraform Engineers

> Beginner to Practical Understanding of Azure Bicep

---

# 📘 What is Bicep?

Bicep is an Infrastructure as Code (IaC) language developed by Microsoft for Azure.

Using Bicep, we can create:

- Virtual Machines
- Storage Accounts
- VNets
- Key Vaults
- App Services
- NSGs
- Azure Policies
- Entire Azure Infrastructure

using code instead of manually creating resources from Azure Portal.

---

# 🔥 Main Difference Between Terraform and Bicep

| Terraform | Bicep |
|---|---|
| Multi-cloud | Azure only |
| Uses HCL language | Uses Bicep DSL |
| Needs state file | No state file |
| External tool | Azure native |
| Uses providers | Uses Azure Resource Types |
| `terraform apply` | `az deployment group create` |
| State management required | Azure handles deployment state |

# 1️⃣ PARAMETERS (`param`)

## Purpose

Used to take input values during deployment.

---

# ✅ Terraform

```hcl
variable "location" {
  default = "East US"
}
```

---

# ✅ Bicep

```bicep
param location string = 'East US'
```

---

# 📖 Syntax Breakdown

```bicep
param <name> <datatype> = <defaultValue>
```

---

# 📌 Example

```bicep
param storageSKU string = 'Standard_LRS'
```

---

# 🧠 Remember

```text
param = deployment input
```

---

# 📌 Common Data Types

| Type | Meaning |
|---|---|
| string | Text |
| int | Integer |
| bool | True/False |
| array | List |
| object | JSON object |

---

# 2️⃣ VARIABLES (`var`)

## Purpose

Used to store calculated or reusable values.

---

# ✅ Terraform

```hcl
locals {
  storage_name = "devstorage"
}
```

---

# ✅ Bicep

```bicep
var storageName = 'devstorage'
```

---

# 📖 Syntax

```bicep
var <name> = <value>
```

---

# 📌 Example

```bicep
var uniqueStorageName = '${storagePrefix}${uniqueString(resourceGroup().id)}'
```

---

# 🧠 Remember

```text
param = external input

var = internal calculation
```

# 3️⃣ RESOURCES (`resource`)

## Purpose

Used to create Azure resources.

This is the MOST IMPORTANT part of Bicep.

---

# ✅ Terraform

```hcl
resource "azurerm_storage_account" "stg" {

}
```

---

# ✅ Bicep

```bicep
resource stg 'Microsoft.Storage/storageAccounts@2025-06-01' = {

}
```

---

# 📖 Syntax

```bicep
resource <symbolicName> '<resourceType>@<apiVersion>' = {

}
```

---

# 📌 Breakdown

| Part | Meaning |
|---|---|
| resource | Keyword |
| stg | Symbolic name |
| Microsoft.Storage | Provider |
| storageAccounts | Resource type |
| 2025-06-01 | API version |

---

# 📌 Real Example

```bicep
resource stg 'Microsoft.Storage/storageAccounts@2025-06-01' = {
  name: 'mystorage123'
  location: 'East US'

  sku: {
    name: 'Standard_LRS'
  }

  kind: 'StorageV2'
}
```

---

# 🧠 Important Concept

## Resource Body = Azure Portal Form

| Azure Portal Field | Bicep Property |
|---|---|
| Name | name |
| Region | location |
| Performance | sku |
| Account Type | kind |

## Purpose

---

# ✅ Terraform

```hcl

  source = "./modules/network"
}
```

---

# ✅ Bicep

```bicep
module network './modules/network.bicep' = {
  name: 'networkDeploy'
}
```

---

# 📖 Syntax

```bicep

}
```

---

# 📌 Example

```bicep

  name: 'storageDeploy'

  params: {
    location: location
  }
}
```

# 🧠 Remember

```text

```

# 8️⃣ DECORATORS

## Purpose

Validation and additional information.

Decorators start with `@`.

---

# 📌 Example

```bicep
@minLength(3)
@maxLength(11)
param storagePrefix string
```

---

# 📌 Common Decorators

| Decorator | Purpose |
|---|---|
| @description | Description |
| @allowed | Allowed values |
| @minLength | Minimum length |
| @maxLength | Maximum length |
| @secure | Hide password |
| @minValue | Minimum number |
| @maxValue | Maximum number |

---

# 📌 Example

```bicep
@secure()
param adminPassword string
```

---

# 🧠 Important

Secure values:
- are not logged
- are not stored in deployment history

# 9️⃣ LOOPS

## Purpose

Create multiple copies of resources/modules.

---

# ✅ Terraform

```hcl
count = 3
```

---

# ✅ Bicep

```bicep
resource stg 'Microsoft.Storage/storageAccounts@2025-06-01' = [
  for i in range(0,3): {
    name: 'storage${i}'
  }
]
```

---

# 📌 Result

```text
storage0
storage1
storage2
```

---

# 🧠 Remember

```text
[for i in range()]
```

means:

```text
Create multiple resources
```

# 🔟 CONDITIONAL DEPLOYMENT

## Purpose

Deploy resource only if condition is true.

---

# 📌 Syntax

```bicep
resource dns 'Microsoft.Network/dnsZones@2023-07-01-preview' = if (deployDns) {

}
```

---

# 📌 Example

```bicep
param deployDns bool

resource dns 'Microsoft.Network/dnsZones@2023-07-01-preview' = if (deployDns) {
  name: 'mydns'
}
```

---

# 🧠 Meaning

```text
If deployDns = true
→ create resource
Else skip
```

# 1️⃣8️⃣ COMPLETE BICEP EXAMPLE

```bicep
metadata description = 'Creates Storage Account'

@description('Azure region')
param location string = 'East US'

@minLength(3)
@maxLength(11)
param storagePrefix string

var uniqueStorageName = '${storagePrefix}${uniqueString(resourceGroup().id)}'

resource stg 'Microsoft.Storage/storageAccounts@2025-06-01' = {
  name: uniqueStorageName
  location: location

  sku: {
    name: 'Standard_LRS'
  }

  kind: 'StorageV2'

  properties: {
    supportsHttpsTrafficOnly: true
  }
}

output storageId string = stg.id
```

# 🔥 Golden Rule For Writing Bicep

Whenever you want to create any Azure resource:

## Think in this order:

```text
1. What inputs are needed?
2. What is the Azure resource type?
3. What properties are required?
4. Any dependencies?
5. Any outputs needed?
```
